# 高并发 IM 聊天室 — 项目汇报 PPT 大纲

> 基于 Reactor 模型 + MySQL + AI 智能助手
> 纯 C 语言实现 · 万级并发 · 零外部框架依赖

---

## 一、项目背景

### 传统聊天室方案的痛点

- 依赖第三方框架（如 Netty、libevent），学习成本高、部署复杂
- 单机模型无法支撑高并发，扩展性差
- 功能单一，缺少 AI 智能交互能力

### 本项目定位

- **从零实现**一套完整的高并发 IM 系统
- 采用业界主流的 **多 Reactor 架构**，对标 Nginx / Redis 的并发模型
- 集成 DeepSeek 大模型，实现**聊天室内嵌 AI 助手**

---

## 二、系统架构总览

```
          ┌──────────────┐
          │   主 Reactor   │  ← epoll 监听新连接
          └──────┬───────┘
     ┌───────────┼───────────┐
     ▼           ▼           ▼
┌─────────┐ ┌─────────┐ ┌─────────┐
│子Reactor │ │子Reactor │ │子Reactor │  ← 各管一组连接
│ (epoll)  │ │ (epoll)  │ │ (epoll)  │
└────┬────┘ └────┬────┘ └────┬────┘
     └───────────┼───────────┘
                 ▼
          ┌──────────────┐
          │   业务线程池    │  ← 解耦 I/O 与业务处理
          └──────┬───────┘
                 ▼
     ┌───────┐ ┌───────┐
     │ MySQL │ │AI API │
     └───────┘ └───────┘
```

**核心设计**：主 Reactor 只负责 accept，子 Reactor 负责读写，线程池处理业务逻辑

## 四、技术亮点：与传统方案的不同

### 1. 多 Reactor + 线程池架构

- **传统**：一个线程处理所有连接（如 select 模型），瓶颈明显
- **本项目**：主 Reactor 管连接、子 Reactor 管 I/O、线程池管业务，**三层解耦**
- 单机支撑 **10240+** 并发连接

### 2. 自定义二进制协议

- **传统**：直接用文本协议（如 JSON over TCP），解析开销大
- **本项目**：8 字节紧凑包头（Magic + Type + Flags + BodyLen），高效序列化
- 环形缓冲区（64KB）+ 状态机解析，支持**半包 / 粘包自恢复**

### 3. AI 大模型原生集成

- **传统**：聊天室仅有基础消息功能
- **本项目**：`/to AI` 直接唤起 DeepSeek 大模型，**无需跳出聊天界面**
- 纯 C 实现，通过 popen + curl 调用 RESTful API，零外部依赖

### 4. 令牌桶限流

- **传统**：简单计数器限流，无法处理突发流量
- **本项目**：令牌桶算法，允许 5 个突发 + 5 个/秒持续，**兼顾体验和防护**

### 5. 终端 UI 气泡渲染

- **传统**：CLI 聊天室只有纯文本输出，阅读困难
- **本项目**：ANSI 色彩 + Unicode 边框气泡，左右对齐，支持**中文 UTF-8 自动换行**
- 多线程互斥 + 非规范终端模式，**接收消息不覆盖正在输入的内容**

### 6. MySQL 持久化

- **传统**：用户数据仅存内存，重启丢失
- **本项目**：MySQL 持久化用户表，密码哈希（SHA256），支持封号管理

## 五、用到的核心技术

| 技术领域 | 具体技术 |
|---------|---------|
| 网络 I/O | epoll + 非阻塞 I/O + MSG_NOSIGNAL |
| 并发模型 | 多 Reactor（主-从） + 业务线程池 |
| 数据缓冲 | 环形缓冲区（ringbuf）64KB × 2 / 连接 |
| 协议设计 | 自定义二进制协议，8B 包头 + 可变包体 |
| 数据库 | MySQL（用户注册 / 登录 / 权限 / 封号） |
| 安全 | 密码 SHA256 哈希 + OpenSSL 库链接 |
| 限流算法 | 令牌桶（Token Bucket） |
| AI 集成 | DeepSeek API（OpenAI 兼容格式，curl 调用） |
| 终端 UI | ANSI 转义序列 + UTF-8 宽字符 + termios 非规范模式 |
| 进程间通信 | eventfd 跨线程唤醒子 Reactor |
