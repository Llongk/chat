# 高并发 IM 聊天室 — 技术执行流程文档

> 本文档以"一条消息从发出到收到"为主线，逐步拆解每个技术的执行流程。

---

## 一、服务器启动流程

服务器启动时按顺序初始化 6 个模块，任何一个失败都会终止启动。

```
main() 入口
  │
  ├─① logger_init("chat_server.log")
  │    打开日志文件，后续所有模块都可以通过 LOG_INFO() 打印日志
  │
  ├─② db_init()
  │    连接 MySQL → 自动建库(chat_db) → 自动建表(users)
  │    如果表里没有管理员 → 自动创建 admin/admin123
  │
  ├─③ setup_signals()
  │    注册信号处理器：Ctrl+C → 优雅停机
  │    忽略 SIGPIPE → 防止对端断开时进程崩溃
  │
  ├─④ client_manager_init()
  │    清空全局客户端表 g_clients[10240]
  │    这张表以 fd 为索引，支持 O(1) 查找
  │
  ├─⑤ threadpool_create(4)
  │    创建 4 个业务工作线程，全部进入 cond_wait 睡眠
  │    等待 reactor 往任务队列里扔任务
  │
  ├─⑥ reactor_init(port=8888, sub_count=4)
  │    创建监听 socket → bind → listen
  │    创建 4 个子 Reactor（每个有自己的 epoll 实例 + eventfd）
  │    创建主 Reactor 线程
  │
  └─⑦ heartbeat_start(timeout=3600, interval=5)
       创建心跳检测线程，每 5 秒醒来检查一次所有客户端
  │
  └─ reactor_run() ← 主 Reactor 开始运行，阻塞在这里
```

**启动后系统的线程分布：**
```
主线程      → 主 Reactor（epoll_wait 监听新连接）
子线程 0~3  → 4 个子 Reactor（各自 epoll_wait 监听已连接客户端）
工作线程 0~3 → 4 个业务线程（cond_wait 等待任务）
心跳线程    → 每 5 秒醒来检查超时
共 10 个线程
```

## 二、TCP 连接建立流程

当一个客户端执行 `connect()` 时：

```
客户端                              服务端
  │                                   │
  │  connect(127.0.0.1:8888)         │
  │ ──────────────────────────────→   │  TCP 三次握手（内核完成）
  │                                   │
  │                          主 Reactor epoll_wait 返回
  │                          发现 listen_fd 可读
  │                                   │
  │                          accept() 得到 client_fd
  │                                   │
  │                          set_nonblock(client_fd)  设为非阻塞
  │                                   │
  │                          client_create(fd)
  │                            ├─ malloc client_t 结构体
  │                            ├─ ringbuf_create(64KB) × 2  (inbuf + outbuf)
  │                            ├─ ratelimit_init() 初始化令牌桶(5个令牌)
  │                            ├─ proto_ctx.state = PARSE_STATE_HEADER
  │                            └─ g_clients[fd] = cli  注册到全局表
  │                                   │
  │                          轮询选择一个子 Reactor (Round-Robin)
  │                          epoll_ctl(ADD, fd, EPOLLIN|EPOLLET)
  │                                   │
  │  连接建立完成，可以发数据了         │
```

**关键点：**
- 主 Reactor **只做 accept**，不处理已连接客户端的读写
- 新连接通过 Round-Robin 轮询分配给子 Reactor
- 每个客户端创建时就分配了 inbuf、outbuf 两个 64KB 的环形缓冲区

## 三、数据发送流程（客户端 → 服务端）

以客户端发送公屏消息"你好"为例：

```
用户输入: 你好
  │
  ├─① 构建 JSON 包体
  │    body = "你好"  (直接作为包体，公屏消息不需要 JSON 封装)
  │
  ├─② 构建包头 (8 字节)
  │    proto_build_header():
  │      magic    = 0x48494D53   ← 固定魔数 "HIMS"
  │      type     = 0x03          ← MSG_PUBLIC_MSG
  │      flags    = 0x00          ← 无特殊标志
  │      body_len = 6             ← "你好" UTF-8 编码 6 字节
  │
  ├─③ 序列化到 packet[]
  │    proto_serialize_header():  包头写入前 8 字节（自动转网络字节序）
  │    memcpy(packet+8, body):    包体紧跟包头
  │
  ├─④ send() 一次性发出
  │    packet = [48 49 4D 53] [03] [00] [00 06] [E4 BD A0 E5 A5 BD]
  │             └─── 包头 ──┘  type flags  len     └── 包体 ──┘
  │    总长度 = 8 + 6 = 14 字节
  │
  └─⑤ 发送完成，数据进入 TCP 发送缓冲区 → 网络传输
```

## 四、数据接收与拆包流程（服务端）

服务端子 Reactor 收到数据后的完整处理链：

```
子 Reactor epoll_wait 返回 → 发现某客户端 fd 可读
  │
  ├─① sub_handle_read()
  │    ET 模式，循环 recv 直到 EAGAIN：
  │
  │    recv(fd, buf, 4096) → 假设收到 14 字节
  │    buf = [48 49 4D 53 03 00 00 06 E4 BD A0 E5 A5 BD]
  │    │
  │    ringbuf_write(cli->inbuf, buf, 14)
  │    这 14 字节原封不动写入 inbuf（包头+包体都在里面）
  │
  │    继续 recv → 返回 EAGAIN → 数据读完，退出循环
  │
  ├─② sub_try_assemble_packet()  ← 拆包核心
  │    while(1) 循环：
  │
  │    ┌─ 状态 PARSE_STATE_HEADER ──────────────────┐
  │    │  readable(14) >= 8?  → YES                   │
  │    │  peek 前 8 字节 → 反序列化包头               │
  │    │  magic == 0x48494D53? → YES ✓               │
  │    │  提取 body_len = 6                           │
  │    │  skip 8 字节 → inbuf 剩 6 字节               │
  │    │  state → PARSE_STATE_BODY                    │
  │    └──────────────────────────────────────────────┘
  │                       ↓
  │    ┌─ 状态 PARSE_STATE_BODY ────────────────────┐
  │    │  readable(6) >= need(6)?  → YES             │
  │    │  malloc(6) 分配包体内存                      │
  │    │  ringbuf_read(inbuf, body, 6)                │
  │    │  body = [E4 BD A0 E5 A5 BD] → "你好"        │
  │    │                                              │
  │    │  创建 task:                                  │
  │    │    task->fd = cli->fd                        │
  │    │    task->msg_type = 0x03 (PUBLIC_MSG)        │
  │    │    task->body = "你好"                       │
  │    │    task->body_len = 6                        │
  │    │                                              │
  │    │  threadpool_add_task(biz_pool,               │
  │    │      chat_service_process, task)             │
  │    │    → cond_signal 唤醒一个工作线程             │
  │    │                                              │
  │    │  state → PARSE_STATE_HEADER（回到包头）       │
  │    └──────────────────────────────────────────────┘
  │
  └─③ 继续 while(1) → readable(0) < 8 → break 退出
```

**如果数据不完整（半包）：**
```
假设只收到 10 字节（包头 8 + 包体只到了 2 字节）：
  PARSE_STATE_HEADER: readable(10) >= 8 → YES → skip 8 → 剩 2
  PARSE_STATE_BODY:   readable(2) >= need(6) → NO → break

  这 2 字节留在 inbuf 里，等下次 recv 补齐到 6 字节再处理
  → 不丢数据，不截断
```

## 五、线程池业务处理流程

工作线程被唤醒后执行 `chat_service_process()`：

```
工作线程从 cond_wait 醒来
  │
  ├─ 从任务队列取出 task
  │    task->msg_type = 0x03 (MSG_PUBLIC_MSG)
  │    task->body     = "你好"
  │    task->fd       = 发送者的 fd
  │
  ├─ switch(msg_type) → handle_public_msg(task)
  │
  ├─ handle_public_msg() 内部：
  │    │
  │    ├─① 令牌桶限流检查
  │    │    ratelimit_check(cli) → 消耗 1 个令牌
  │    │    令牌够 → 继续处理
  │    │    令牌不够 → 返回 ERR_RATE_LIMITED，拒绝发送
  │    │
  │    ├─② 构建广播 JSON
  │    │    {"from":"qhl","msg":"你好","is_admin":0,"online":3}
  │    │
  │    ├─③ 遍历所有在线客户端 (client_foreach)
  │    │    对每个在线客户端调用:
  │    │      chat_send_message(cli->fd, MSG_PUBLIC_BROADCAST, body)
  │    │
  │    └─④ 更新发送者活跃时间
  │         client_update_active(cli) → last_active = now
  │
  ├─ free(task->body)  释放包体
  ├─ free(task)         释放任务节点
  │
  └─ 回到 cond_wait 等待下一个任务
```

## 六、消息发送回客户端流程

`chat_send_message()` 负责把响应写回客户端：

```
chat_send_message(target_fd, MSG_PUBLIC_BROADCAST, json_body)
  │
  ├─① 计算总长度
  │    total_len = 8(包头) + body_len(包体)
  │
  ├─② 组装完整包
  │    proto_build_header()   → 填充 magic/type/body_len
  │    proto_serialize_header() → 包头写入 packet 前 8 字节
  │    memcpy(packet+8, body) → 包体紧跟包头
  │
  ├─③ 写入目标客户端的 outbuf
  │    pthread_mutex_lock(outbuf_mutex)   ← 加锁（多线程安全）
  │    ringbuf_write(cli->outbuf, packet, total_len)
  │    pthread_mutex_unlock(outbuf_mutex) ← 解锁
  │
  └─④ 通知子 Reactor 发送
       chat_notify_sub_reactor(cli->sub_reactor_idx)
         → write(eventfd, &val, 8)   ← 往 eventfd 写 1 唤醒子 Reactor
```

**子 Reactor 被 eventfd 唤醒后：**
```
子 Reactor epoll_wait 返回 → eventfd 可读
  │
  ├─ sub_handle_write(cli)
  │    │
  │    ├─ ringbuf_peek(outbuf, buf, 4096)  ← 从 outbuf 取数据
  │    │
  │    ├─ send(fd, buf, len, MSG_NOSIGNAL) ← 发送到网络
  │    │    成功 → ringbuf_skip(outbuf, n)  ← 消费已发送的字节
  │    │    EAGAIN → break，等下次 EPOLLOUT
  │    │
  │    ├─ 还有剩余数据？
  │    │    YES → 注册 EPOLLOUT，等下次可写再继续发
  │    │    NO  → 移除 EPOLLOUT，只保留 EPOLLIN
  │    │
  │    └─ pthread_mutex_unlock(outbuf_mutex)
```

## 七、客户端接收与显示流程

客户端 recv 线程收到数据后：

```
接收线程 recv_thread()
  │
  ├─① recv() 读数据到 buf
  │    buf 中: [包头8B][包体N字节]
  │
  ├─② 拆包循环（和服务器类似，但用栈上 buf 而非 ringbuf）
  │    while (buf_len >= 8):
  │      peek 包头 → 校验 magic
  │      body_len 不够 → break 等下次
  │      取出包体 → body[hdr.body_len] = '\0'
  │
  ├─③ switch(hdr.type) 分发处理
  │    MSG_PUBLIC_BROADCAST → handle_public_broadcast(body)
  │      │
  │      ├─ 从 JSON 提取 from、msg、is_admin
  │      ├─ convert_newlines(msg) → 把 \n 转为真正换行
  │      ├─ 判断 from == g_username？
  │      │    YES → show_self_public()    右对齐青色气泡
  │      │    NO  → show_other_public()   左对齐绿色气泡
  │      └─ 显示在终端
  │
  └─④ 继续循环处理下一条消息
```

## 八、环形缓冲区 (Ring Buffer) 工作原理

```
创建时: capacity = 64KB
        [                                    ]
        ↑                                    ↑
     read_pos=0                          write_pos=0
     readable = 0

写入 100 字节后:
        [==data(100B)==                       ]
        ↑                 ↑
     read_pos=0       write_pos=100
     readable = 100

peek 8 字节（看不搬走）:
        [==data(100B)==                       ]
        ↑                 ↑
     read_pos=0       write_pos=100     ← 指针不变

skip 8 字节（消费掉）:
        [........==data(92B)==              ]
               ↑            ↑
          read_pos=8    write_pos=100
          readable = 92

写满一圈后（write_pos 绕回开头）:
        [new_data........old_data.........]
                    ↑              ↑
              read_pos         write_pos
              （环形回绕，write_pos < read_pos 也可读）
```

**关键操作：**
| 操作 | 做什么 | 移动指针 |
|------|--------|---------|
| `write` | 数据写入缓冲区 | write_pos 前进 |
| `peek` | 读取数据但**不消费** | 不动 |
| `skip` | 消费数据（丢弃） | read_pos 前进 |
| `read` | 读取数据**并消费** | read_pos 前进 |
| `readable` | 计算可读字节数 | 不动 |

## 九、epoll ET 模式

```
LT 模式 (Level-Triggered, 默认):
  fd 可读 → epoll_wait 返回 → 处理 → fd 仍然可读 → epoll_wait 再次返回
  → 可能反复触发，浪费 CPU

ET 模式 (Edge-Triggered):
  fd 从不可读变为可读 → epoll_wait 返回一次
  → 必须一次性读完所有数据（循环 recv 直到 EAGAIN）
  → 否则不会再通知，数据就"漏"了
```

**项目中 ET 模式的使用：**

```c
// sub_handle_read() — 必须循环读到 EAGAIN
while (1) {
    ssize_t n = recv(cli->fd, buf, sizeof(buf), 0);
    if (n > 0) {
        ringbuf_write(cli->inbuf, buf, n);  // 读到数据
    } else if (n == 0) {
        cli->alive = 0;  break;              // 对端关闭
    } else {
        if (errno == EAGAIN) break;           // 读完了
        cli->alive = 0;  break;              // 错误
    }
}
```

## 十、令牌桶限流流程

```
初始化（客户端连接时）:
  capacity = 5, rate = 5/秒
  tokens = 5 (满桶)
  last_refill = 当前时间

每次发消息时:
  │
  ├─ 计算时间差: elapsed = now - last_refill
  │
  ├─ 补充令牌: tokens += elapsed × rate
  │    上限: tokens = min(tokens, capacity)
  │
  ├─ 判断:
  │    tokens >= 1.0 → tokens -= 1.0 → return 0 (放行)
  │    tokens <  1.0 → return -1 (拒绝)
  │
  └─ 更新: last_refill = now

示例（连续快速发 6 条）:
  消息1: elapsed=0.01s → 补 0.05 → tokens=5.05→cap=5 → 消耗1 → 4.0 ✓
  消息2: elapsed=0.01s → 补 0.05 → tokens=4.05        → 消耗1 → 3.05 ✓
  消息3: elapsed=0.01s → 补 0.05 → tokens=3.10        → 消耗1 → 2.10 ✓
  消息4: elapsed=0.01s → 补 0.05 → tokens=2.15        → 消耗1 → 1.15 ✓
  消息5: elapsed=0.01s → 补 0.05 → tokens=1.20        → 消耗1 → 0.20 ✓
  消息6: elapsed=0.01s → 补 0.05 → tokens=0.25        → 不够1 → ✗ 拒绝
```

## 十一、心跳检测流程

```
服务端心跳线程（每 5 秒一轮）:
  │
  ├─ sleep(5) → 醒来
  │
  ├─ client_foreach(hb_check_callback)
  │    遍历每个在线客户端：
  │      now - cli->last_active > 3600?
  │        YES → cli->alive = 0  (标记不活跃)
  │        NO  → 跳过
  │
  └─ 继续 sleep(5)

客户端心跳线程（每 10 秒一轮）:
  │
  ├─ sleep(10) → 醒来
  │
  ├─ send_packet(MSG_HEARTBEAT_REQ, ...)
  │    发送心跳 ping
  │
  └─ 继续 sleep(10)

服务端收到心跳:
  │
  ├─ handle_heartbeat(task)
  │    ├─ client_update_active(cli) → last_active = now  ← 续命
  │    └─ 回复 pong
```

**心跳如何防止僵尸连接：**
```
正常情况:
  客户端每 10 秒发 ping → 服务端更新 last_active → 永远不超时

客户端断网/崩溃:
  客户端无法发 ping → last_active 不更新
  → 3600 秒后心跳线程检测到超时 → cli->alive = 0
  → 子 Reactor 下次循环发现 alive=0 → close(fd) → 清理资源
```

## 十二、MySQL 数据库操作流程

以登录为例：

```
handle_login(task)
  │
  ├─① 解析 JSON 包体
  │    {"username":"qhl","password":"123"}
  │    提取 username = "qhl"
  │    提取 password = "123"
  │
  ├─② 调用 db_user_login(username, password)
  │    │
  │    ├─ db_escape_string(username)  ← 防 SQL 注入
  │    │
  │    ├─ SHA256(password) → 哈希值
  │    │    "123" → "a665a45920422f9d417e4867efdc4fb8a04a1f3fff1fa07e998e86f7f7a27ae3"
  │    │
  │    ├─ SQL 查询:
  │    │    SELECT password_hash, is_admin, status FROM users
  │    │    WHERE username = 'qhl'
  │    │
  │    ├─ 比较哈希值:
  │    │    存储的哈希 == 计算出的哈希？
  │    │    YES → 登录成功
  │    │    NO  → 密码错误
  │    │
  │    ├─ 检查 status:
  │    │    status == 0? → 账号被封禁，拒绝登录
  │    │
  │    └─ 更新最后登录时间:
  │         UPDATE users SET last_login = NOW() WHERE username = 'qhl'
  │
  ├─③ 登录成功:
  │    client_set_logged_in(cli, username) → cli->logged_in = 1
  │    如果 is_admin → cli->is_admin = 1
  │
  └─④ 发送响应给客户端
       {"status":"ok","username":"qhl","is_admin":0}
```

## 十三、优雅停机流程

用户按 Ctrl+C 后：

```
SIGINT 信号 → signal_handler()
  │
  ├─ g_running = 0
  ├─ g_reactor.running = 0
  │
  ├─ shutdown(listen_fd)       ← 关闭监听 socket
  │    → 主 Reactor epoll_wait 立即返回 → 退出循环
  │
  ├─ write(eventfd) × 4       ← 唤醒所有子 Reactor
  │    → 子 Reactor epoll_wait 立即返回 → 退出循环
  │
  主线程恢复执行:
  │
  ├─ heartbeat_stop()          ← 停止心跳线程
  │    hb->running = 0
  │    pthread_join(hb->thread) ← 等待心跳线程退出
  │
  ├─ reactor_stop()            ← 停止所有 Reactor
  │    每个子 Reactor:
  │      running = 0
  │      write(eventfd) 唤醒
  │      pthread_join(sub->thread) ← 等待退出
  │    遍历所有客户端 → close(fd)
  │
  ├─ threadpool_destroy()      ← 停止业务线程池
  │    shutdown = 1
  │    cond_broadcast() ← 唤醒所有工作线程
  │    每个工作线程:
  │      发现 shutdown → break → 退出循环
  │    pthread_join × 4 ← 等待全部退出
  │
  ├─ client_manager_destroy()  ← 释放所有客户端资源
  │    遍历 g_clients[]:
  │      ringbuf_destroy(inbuf/outbuf)
  │      pthread_mutex_destroy(outbuf_mutex)
  │      free(cli)
  │
  ├─ db_close()                ← 关闭 MySQL 连接
  │
  └─ logger_close()            ← 关闭日志文件
       LOG_INFO("Server stopped. Goodbye!")
```

## 十四、完整消息流总览

一条公屏消息"你好"从发出到所有人看到的完整路径：

```
  Alice                    服务端                         Bob
    │                        │                             │
    │ 输入 "你好"             │                             │
    │                        │                             │
    │ send([包头][你好])      │                             │
    │ ────────────── TCP ──→ │                             │
    │                        │                             │
    │              ① recv → inbuf                          │
    │              ② 拆包: peek包头 + read包体              │
    │              ③ 创建 task → 线程池                     │
    │              ④ 令牌桶检查 → 通过                      │
    │              ⑤ 构建广播 JSON                         │
    │              ⑥ client_foreach:                       │
    │                 ├─ chat_send_message(Alice)           │
    │                 │   包头+JSON → Alice.outbuf          │
    │                 │   eventfd → 唤醒 Alice 的子Reactor  │
    │                 │   send(Alice) → Alice 看到自己的消息 │
    │                 │                                     │
    │                 └─ chat_send_message(Bob)             │
    │                     包头+JSON → Bob.outbuf            │
    │                     eventfd → 唤醒 Bob 的子Reactor    │
    │                     send(Bob) ──── TCP ──→            │
    │                                               ⑦ recv → buf
    │                                               ⑧ 拆包
    │                                               ⑨ 显示气泡
    │                                                       │
    │  ←────── TCP ──── send(Alice)                         │
    │ ⑩ recv → 拆包                                        │
    │ ⑪ 显示右对齐气泡                                      │
```
